= Scopri il supporto e le funzionalità ONTAP per gli host Proxmox
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Le funzionalità supportate per la configurazione host con NVMe over Fabrics (NVMe-oF) variano in base alla versione di ONTAP e Proxmox.

[cols="30,20,20"]
|===
| Funzione ONTAP | Versione host Proxmox | Versione ONTAP 


| NVMe/TCP è una funzionalità aziendale completamente supportata  a| 
9.0
| 9.10.1 o successivo 


| Fornisce dettagli ONTAP per gli spazi dei nomi NVMe/FC e NVMe/TCP utilizzando il nativo `nvme-cli` pacchetto.  a| 
8.0
| 9.10.1 o successivo 


| Il traffico NVMe e SCSI è supportato sullo stesso host utilizzando NVMe multipath per gli spazi dei nomi NVMe-oF e dm-multipath per le LUN SCSI  a| 
8.0
| 9.4 o successivo 
|===
ONTAP supporta le seguenti funzionalità host SAN indipendentemente dalla versione ONTAP in esecuzione sulla configurazione del sistema.

[cols="70,30"]
|===
| Caratteristica | Versione host Proxmox 


| IL `nvme-cli` il pacchetto include script di connessione automatica che eliminano la necessità di script di terze parti  a| 
9.0



| La regola udev nativa nel  `nvme-cli` pacchetto fornisce bilanciamento del carico round-robin per il multipathing NVMe  a| 
9.0



| Il multipathing NVMe nativo è abilitato per impostazione predefinita  a| 
8.0

|===

NOTE: Per i dettagli sulle configurazioni supportate, vederelink:https://mysupport.netapp.com/matrix/["Tool di matrice di interoperabilità"^] .
