= Scopri di più sul supporto e sulle funzionalità ONTAP per SUSE Linux Enterprise Server
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Le funzionalità supportate per la configurazione host con NVMe over Fabrics (NVMe-oF) variano in base alla versione di ONTAP e SUSE Linux Enterprise Server.

[cols="50,25,25"]
|===
| Caratteristica | Versione host di SUSE Linux Enterprise Server | Versione ONTAP 


| La crittografia Transport Layer Security (TLS) 1.3 è supportata per NVMe/TCP  a| 
15 SP6 o successivo
| 9.16.1 o successivo 


| L'autenticazione in-band sicura è supportata tramite NVMe/TCP tra un host RHEL e un controller ONTAP  a| 
15 SP4 o successivo
| 9.12.1 o successivo 


| I controller di rilevamento persistenti (PDC) sono supportati utilizzando un NQN di rilevamento univoco  a| 
15 SP4 o successivo
| 9.11.1 o successivo 


| NVMe/TCP fornisce spazi dei nomi utilizzando il nativo `nvme-cli` pacchetto  a| 
15 SP4 o successivo
| 9.10.1 o successivo 


| Il traffico NVMe e SCSI è supportato sullo stesso host utilizzando NVMe multipath per gli spazi dei nomi NVMe-oF e dm-multipath per le LUN SCSI  a| 
15 SP1 o successivo
| 9.4 o successivo 
|===
ONTAP supporta le seguenti funzionalità host SAN indipendentemente dalla versione ONTAP in esecuzione sulla configurazione del sistema.

[cols="75,25"]
|===
| Caratteristica | Versione host di SUSE Linux Enterprise Server 


| L'avvio SAN è abilitato tramite il protocollo NVMe/FC  a| 
15 SP7 o successivo



| Il multipathing NVMe nativo è abilitato per impostazione predefinita  a| 
15 SP1 o successivo



| IL `nvme-cli` il pacchetto include script di connessione automatica che eliminano la necessità di script di terze parti  a| 
15 SP1 o successivo

|===

NOTE: Per i dettagli sulle configurazioni supportate, vederelink:https://mysupport.netapp.com/matrix/["Tool di matrice di interoperabilità"^] .
